# Security Policy

## サポート対象

| 対象 | サポート |
| --- | --- |
| `main` ブランチの最新状態 | Yes |
| 最新の公開リリース | Yes |
| それ以前の古いリリース | No |

## 脆弱性の報告方法

脆弱性と思われる内容は、公開 `Issue`、公開 `Discussion`、公開 `Pull Request` では報告しないでください。

このリポジトリでは GitHub の private vulnerability reporting を利用できます。まずはリポジトリの `Security` タブから非公開で報告してください。

もし GitHub 上で private report を作成できない場合は、`@Freeesia` へ GitHub 上で非公開に連絡してください。公開 issue での連絡は避けてください。

## 報告時に含めてほしい情報

- 影響を受けるバージョン、タグ、またはコミット
- 想定される影響
- 再現手順
- 必要ならスクリーンショット、ログ、サンプル設定
- 既知の回避策があればその内容

## 対応方針

- まず受領確認を行います
- 内容を確認し、必要に応じて追加情報を依頼します
- 修正が必要な場合は、非公開で修正と公開準備を進めます

## 対象になりやすい領域

- Google API キーや OAuth トークンの取り扱い
- `%LocalAppData%/StudioFreesia/WondayWall/` 配下の設定や履歴の保存
- 生成画像やローカルファイルの扱い
- インストーラーや更新導線
