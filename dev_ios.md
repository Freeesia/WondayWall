# WondayWall iOS版 開発仕様

## 目的

WondayWall iOS版は、ユーザーの予定・ニュース・ユーザープロンプトをもとに、iPhone向けの壁紙候補画像を生成するアプリである。

iOSでは通常のApp Storeアプリからホーム画面・ロック画面の壁紙を直接変更できないため、アプリは壁紙の自動適用を行わない。生成した画像をアプリ内に保存し、写真ライブラリ保存、共有、通知、壁紙設定手順の表示によって、ユーザーが手動で壁紙に設定できる状態にする。

## 基本方針

- 手動生成とバックグラウンド定期生成を提供する。
- 生成処理は `GenerationCoordinator` に集約する。
- 壁紙関連の処理は `WallpaperService` に集約する。
- スケジュール設定は `RunsPerDay`、つまり「1日あたりの自動生成回数」で管理する。
- バックグラウンド定期生成はベストエフォートとし、指定時刻ぴったりの実行は保証しない。
- 自前の中間サーバーは用意しない。
- Google AI画像生成のjobId復旧は行わない。
- 通信断、アプリ停止、OS中断、生成失敗時は失敗履歴を残し、次回以降に再生成できるようにする。

## iOSで提供すること

- Google AIによる壁紙候補画像の生成
- Google Calendarの予定を使った生成コンテキスト作成
- RSS/ニュースを使った生成コンテキスト作成
- ユーザープロンプトを使った生成コンテキスト作成
- 手動生成
- バックグラウンド定期生成
- アプリ内画像保存
- 生成履歴保存
- 写真ライブラリ保存
- 共有シート表示
- 壁紙設定手順の表示
- 生成完了通知
- 生成失敗通知

## iOSで提供しないこと

- 壁紙の自動適用
- jobIdによる生成結果の後追い取得
- 自前中間サーバー
- iCloud同期
- ウィジェット
- Live Activity
- MDM向け壁紙設定

## スケジュール仕様

### RunsPerDay

`RunsPerDay` は、1日に自動生成を試みる回数を表す。

```text
RunsPerDay = 1日あたりの自動生成回数
```

`RunsPerDay` は分単位の更新間隔ではない。例えば `RunsPerDay = 4` の場合は、1日を4つのスロットに分け、各スロットが未処理なら生成を試みる。

### 選択肢

```text
1, 2, 3, 4, 6, 8, 12, 24
```

表示例:

```text
1回/日
2回/日
3回/日
4回/日
6回/日
8回/日
12回/日
24回/日
```

### スロット間隔

| RunsPerDay | スロット間隔 |
|---:|---:|
| 1 | 24時間 |
| 2 | 12時間 |
| 3 | 8時間 |
| 4 | 6時間 |
| 6 | 4時間 |
| 8 | 3時間 |
| 12 | 2時間 |
| 24 | 1時間 |

### 定期生成の判定

定期生成では、現在時刻に対応するスケジュールスロットがすでに処理済みかを確認する。

```text
1. RunsPerDay から当日のスケジュールスロット一覧を作る
2. 現在時刻以前の最新スロットを取得する
3. そのスロット以降に成功履歴があれば生成しない
4. そのスロット以降に成功履歴がなければ生成を試みる
```

擬似コード:

```text
latestSlot = GetLatestScheduledSlotAtOrBefore(now, runsPerDay)
lastSuccessfulGeneratedAt = 最後に成功した生成時刻

if lastSuccessfulGeneratedAt >= latestSlot:
    skip
else:
    generate
```

### 実行保証

バックグラウンド定期生成はiOSのバックグラウンド実行制御に依存する。

以下の影響を受ける。

- 低電力モード
- 通信状態
- 電源状態
- アプリの利用状況
- OSのバックグラウンド実行判断

ユーザー向け文言は以下の意味にする。

```text
1日あたりの自動更新回数を指定します。
未処理の生成タイミングがあれば生成を試みます。
iOSの制御により、実行時刻は前後する場合があります。
```

## 生成モード

### 手動生成

ユーザー操作で即時生成する。

```text
1. ユーザーが「今すぐ生成」を押す
2. GenerationCoordinatorを呼ぶ
3. ContextServiceが生成コンテキストを作る
4. GoogleAiServiceが画像を生成する
5. 画像をアプリ内に保存する
6. HistoryServiceが履歴を保存する
7. 画面を更新する
```

### バックグラウンド定期生成

バックグラウンド定期生成は、iOSの `BGTaskScheduler`、アプリ起動時、フォアグラウンド復帰時に実行可否を確認する。

```text
1. BGTaskScheduler、アプリ起動時、フォアグラウンド復帰時にチェックする
2. RunsPerDayから未処理スロットを判定する
3. 未処理スロットがなければスキップする
4. 未処理スロットがあれば1回だけ生成する
5. 成功・失敗・スキップを履歴に保存する
6. 必要に応じて通知する
7. 次回BGTaskを登録する
```

## サービス構成

### AppConfigService

設定の読み書きを行う。

主な設定:

```text
- Google AI設定
- Google Calendar設定
- RSSソース
- ユーザープロンプト
- RunsPerDay
- 自動生成有効/無効
- 変化がなければスキップ
- 写真保存設定
- 通知設定
- Wi-Fiのみ生成
- 低電力モード時は生成しない
```

### ContextService

画像生成に使う文脈を作る。

取得対象:

- Google Calendarの予定
- RSS/ニュース
- ユーザープロンプト

### GoogleAiService

Google AIに画像生成を依頼し、レスポンスとして受け取った画像を保存する。

方針:

- jobId方式は使わない。
- レスポンス受信後、即座にローカル保存する。
- 保存前に失敗した場合は生成失敗扱いにする。

### GenerationCoordinator

生成処理全体を統括する。

責務:

- 多重実行防止
- 手動生成
- スケジュールスロット判定
- コンテキスト取得
- 画像生成
- 画像保存
- `WallpaperService` 呼び出し
- 通知依頼
- 履歴保存

### WallpaperService

生成画像を壁紙として利用できる状態にするサービス。

現時点のiOSでは通常アプリから壁紙を直接変更できないため、`WallpaperService` は写真ライブラリ保存、共有、壁紙設定手順の表示を行う。

将来iOSで通常アプリから壁紙設定できる公開APIが提供された場合は、この `WallpaperService` に直接適用処理を追加する。

責務:

- 生成画像の存在確認
- 壁紙としての利用可能性確認
- 写真ライブラリへの保存
- 共有シート表示
- 壁紙設定手順の表示
- 将来APIが利用可能になった場合の壁紙直接適用

### HistoryService

生成履歴を保存する。

保存する情報:

```text
- 実行日時
- 成功/失敗/スキップ
- 画像パス
- 使用した予定
- 使用したニュース
- 使用したプロンプト
- エラー概要
- サービス種別
```

### NotificationService

通知を扱う。

通知種別:

- 生成完了通知
- 生成失敗通知

通知タップ時は、最新の生成画像詳細または該当履歴詳細を開く。

### BackgroundTaskService

バックグラウンド定期生成を扱う。

責務:

- BGTaskSchedulerへの登録
- BGTask実行時の生成可否判定
- アプリ起動時・フォアグラウンド復帰時の生成可否判定
- 次回BGTaskの登録

## 権限

### 通知権限

生成完了・生成失敗通知に使う。

バックグラウンド定期生成を有効にしたタイミング、または通知設定を有効にしたタイミングで要求する。

### 写真ライブラリ追加権限

生成画像を写真ライブラリに保存するために使う。

写真全体の読み取りは要求せず、追加用途を優先する。

### Google OAuth

Google Calendarの予定取得に使う。

取得した予定は生成コンテキストに反映する。

### ネットワーク

以下に使う。

- Google AI画像生成
- Google Calendar取得
- RSS取得

## 失敗時の扱い

### 生成失敗

```text
- 履歴に失敗を保存する
- エラー概要を保存する
- 必要なら通知する
- 次回再生成できるようにする
```

### スキップ

以下の場合は生成せず、スキップ扱いにする。

- 現在のスケジュールスロットがすでに処理済み
- 変化がなければスキップ設定が有効で、生成材料に変化がない

### 通信断・OS中断

```text
- 生成失敗扱いにする
- jobIdによる後追い取得はしない
- 次回実行時に再生成する
```

## 最終仕様

WondayWall iOS版は、`RunsPerDay` による1日あたりの自動生成回数を持つ。

バックグラウンド定期生成では、現在時刻に対応するスケジュールスロットが未処理の場合のみ1回生成する。

iOSでは現時点で壁紙の自動適用ができないため、生成後はアプリ内保存、写真保存、共有、通知、壁紙設定手順の提示までを行う。これらは `WallpaperService` の責務とし、将来APIが利用可能になった場合は同サービス内で直接適用にも対応する。

通信断やOS中断時の生成結果復旧は行わず、失敗履歴を残して次回再生成する。
